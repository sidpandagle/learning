The paradox of unlimited options

Before you built this system, the problem was learning too little. Now there's a different risk: learning the wrong things — spending your careful, well-structured sessions on topics that don't actually move you forward.

There is always more to learn than time allows. Every subject branches into sub-subjects. Every skill reveals prerequisites you didn't know you were missing. Without a prioritisation method, motivated learners often drift — picking the most interesting topic at the moment rather than the most valuable one over time.

The goal of this lesson is a simple framework to answer: given everything I could learn, what should I work on next?

Not all knowledge has the same value

Some things you learn are used once. Some are used every day. Some unlock dozens of other things. Some are interesting but isolated.

Think of knowledge as having two dimensions: how broadly it applies (does it help in one situation or many?) and how long it stays useful (is it still true in five years?). The most valuable things to learn score high on both.

The value matrix

Learn first
Broadly applicable + durable

Fundamentals, mental models, core skills in your field, learning-to-learn itself. High leverage — each hour compounds.

Learn when needed
Narrowly applicable + durable

Deep specialist knowledge. Valuable if it's your domain; wasteful otherwise. Learn it when the problem demands it.

Worth the time
Broadly applicable + fades

Current tools, popular frameworks, timely techniques. Useful now but likely obsolete. Learn what you need, don't over-invest.

Skip or defer
Narrowly applicable + fades

Trivia, hyper-specific syntax for a tool you rarely use, trending topics you have no use for. Learn only if you enjoy it.

High-leverage knowledge

Some things you learn make everything else easier to learn. These are called high-leverage topics — knowledge that acts as a multiplier on future learning.

Examples of high-leverage knowledge:

  • ↑ How memory and learning work — what you've been studying in this course. Applies to every subject you ever study.
  • ↑ First principles in your field — the foundational laws or concepts from which everything else in a domain is derived.
  • ↑ Core communication skills — writing, explaining clearly, asking good questions. Used in every context for life.
  • ↑ Mathematics and statistics fundamentals — unlocks reasoning in science, economics, technology, and data.
  • ↑ Mental models from multiple disciplines — frameworks for thinking that apply across domains (probability, incentives, systems thinking).
The compounding principle High-leverage knowledge compounds. If you understand how memory works, every future study session is more effective. If you understand the first principles of your field, every new concept in that field clicks faster. Invest in foundations before polish. (Scott H. Young — Ultralearning)

The three-question filter

When you have a list of things you want to learn and need to decide what to do first, run each item through these three questions in order. Any item that fails a question moves down the list.

1

Will I actually use this?

Not "could I use this someday" — but "will I use this in the next 3 months?" Learning is wasted if it's never applied. Things with no near-term application decay fast even with spaced repetition. If the answer is no, defer it until you have a concrete use case.

2

Does knowing this unlock other things I want to learn?

Prerequisites compound. If learning X means Y and Z become easier, learn X first. If learning X is a dead end that connects to nothing else, it might still be worth it — but it's not a priority over something that unlocks multiple paths.

3

Am I the right person to learn this, and is now the right time?

Some knowledge is better learned from experience than study. Some topics need a trigger — a real problem — to stick. If you could learn this better by doing a project than by reading, do the project. If there's no active context to absorb it, the timing is probably wrong.

The "interesting" trap Interesting is not the same as useful. The brain is naturally drawn to novel, stimulating topics — this is why people spend hours learning about things they never apply. Interesting is a good signal for engagement, but it's not a substitute for the three-question filter. Run interesting topics through the filter too.

How to handle a long list

If you have ten things you want to learn, the goal isn't to pick one and abandon the rest — it's to sequence them. Some things will naturally come before others (prerequisites). Some will become relevant only after you've started working on something specific.

A simple approach that works:

  1. Write the full list — everything you want to learn, no filtering yet.
  2. Score each item on three dimensions: how applicable (1–3), how durable (1–3), and how much it unlocks (1–3). Add them up.
  3. Work on the top scorer first. Keep the rest of the list somewhere visible so the brain stops worrying about losing them.
  4. Reassess every 4–6 weeks — your situation changes, and so does what's valuable to learn next.

What comes next

This is the end of the foundational curriculum — but it's the beginning of the practice. The system only works if you use it. The next step is not another lesson. It's running the system on something real.

Pick one subject you actually want to learn. Apply the prioritisation framework to confirm it's the right choice. Set up your habit stack. Start making cards. Take atomic notes. Review them. Do that for four weeks, and you'll have more evidence about how you learn than any lesson could give you.
