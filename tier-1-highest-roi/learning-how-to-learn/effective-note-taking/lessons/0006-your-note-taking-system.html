Your Note-Taking System

All the pieces exist. Now build the system that makes them automatic — and know when it's actually working.

What you'll learn

  • Where to keep notes — and why one place beats many
  • The minimum viable folder structure that works for any context
  • The 10-minute weekly review that keeps your system alive
  • Three tests to know if your system is genuinely working
  • How to make note-taking stick as a habit, not a chore

The tool trap

Before building your system, one warning: most people spend more energy choosing and switching tools than they spend taking notes. Notion vs. Obsidian vs. Roam vs. a paper notebook. This is a distraction.

The tool is nearly irrelevant. The practices — Cornell structure, the pause, processing levels, connections — work identically across all of them. Choose one tool and stay there for at least 3 months before reconsidering.

Notion
Best for: structured, visual organisation
Flexible databases, good templates. Slightly slower for quick capture. Free tier covers most needs.
Obsidian
Best for: linking ideas, long-term depth
Local files, powerful linking, best PKM ecosystem. Steeper learning curve. Free for personal use.
Apple Notes / Notepad
Best for: starting immediately
Zero friction. Already installed. Surprisingly capable for Cornell + book summaries. Often underestimated.
Paper notebook
Best for: retention and focus
Research suggests handwriting improves retention. No distractions. Cornell works perfectly on paper. Retrieval is harder.
The recommendation Start with whatever is least friction — probably something you already use. Build the habits first. Switch tools only if the tool becomes the bottleneck, which is rarely the real problem.

A minimal folder structure

Over-organising notes is a form of procrastination. You don't need tags, nested folders, and PARA systems from day one. You need three folders:

📁 Notes
📁 Inbox
All new captures go here first
Processed during weekly review
📁 Reference
Processed notes, book summaries, glossaries
Things you'll return to
📁 Archive
Old notes, completed projects, irrelevant captures
Out of sight, not deleted

Everything enters the Inbox. The weekly review processes Inbox into Reference or Archive. Reference is where your system lives. The Archive keeps things without cluttering your view.

Add sub-folders inside Reference only when you have 10+ notes on a topic. Don't create structure before you have enough notes to justify it.

The 10-minute weekly review

Systems die without maintenance. The weekly review is what keeps yours alive. It should take 10 minutes — no more. Do it on the same day each week so it becomes automatic.

3 min

Clear the Inbox. Process every note in the Inbox: complete any Level 1 gaps, add cues if needed, file to Reference or Archive. Delete anything that didn't survive the week.

3 min

Skim Reference notes from this week. Quickly re-read anything you filed in the last 7 days. Add any missing connections you notice now that you have more distance.

2 min

Check for stranded actions. Any action items from meeting notes that haven't been moved to your task system? Move them now.

2 min

Ask one question. "What's the most useful thing I captured this week?" Spend 2 minutes thinking about where it connects to what you're working on. This habit over time produces genuine insight.

Three tests: is your system working?

You don't need to track metrics. You need three yes/no tests, run monthly:

🔍

The retrieval test. Think of something you've learned in the last 3 months. Can you find the relevant note in under 2 minutes? If not: your organisation or search is the bottleneck.

🧠

The memory test. Pick a note from 4 weeks ago. Before reading it, try to recall what it says. If the note surprises you, your processing was insufficient. If it confirms memory, your system is working.

⚡

The friction test. The next time you want to take a note, notice whether you hesitate. Hesitation means the system is too complex. Notes should feel like zero effort to start.

Making it a habit

The system you built across these 6 lessons has two main habits to anchor:

  1. Capture + same-day process. Every time you finish a learning session (book chapter, meeting, video), you process the same day. Attach it to the end of the session: "The session ends when I've done Level 1 processing."
  2. Weekly review. Attach it to an existing weekly anchor: Friday end-of-day, Sunday morning, Monday startup. Same time, same day. Set a 10-minute calendar block. Protect it.

Don't try to do both perfectly from week one. Start with same-day processing. Add the weekly review once that's automatic. Add connections (Level 2) once the review is automatic. Build in layers. (James Clear — Habit Stacking)

Your one action from this lesson Set up your three folders (Inbox / Reference / Archive) in whatever tool you'll use. Move any existing notes into the right folders. Then set a recurring 10-minute block in your calendar for the same time each week. That's your system. Everything else follows from practice.
Final Self-Check

Test what you just learned

1. You're trying to decide between Notion and Obsidian for your note-taking system. You've spent two hours reading comparisons. What's the most useful thing to do next?

A Read one more in-depth comparison to make sure you choose the right tool
B Pick one — whichever has least friction — and commit to it for 3 months before reconsidering
C Use both in parallel to figure out which fits better over time
D Start with paper until you're confident in the habits, then migrate to digital
Correct. Two hours of tool comparison is already too much. The tool is not the bottleneck — the habits are. Pick whichever has least friction and start taking notes today. You can always switch later.
The tool matters far less than the habits. The practices — Cornell structure, the pause, processing, connections — work in any tool. Pick one with least friction and commit for 3 months. Spending more time comparing tools is procrastination in disguise.

2. You have 50 processed notes across 8 different topics. Your Reference folder has no sub-folders. Should you add them?

A Yes — 8 topics is already enough to warrant organised sub-folders
B Only if a specific topic has 10+ notes — don't create structure before the volume justifies it
C Yes — sub-folders are always better than one flat folder for retrieval speed
D No — never use sub-folders, rely on search instead
Exactly. Create structure when volume demands it, not in anticipation of it. If most topics have 3–5 notes, search is faster than navigating sub-folders. Add sub-folders only when you find yourself scrolling past 10+ notes on a topic looking for the right one.
Add sub-folders when a topic has 10+ notes — not before. Pre-emptive organisation is overhead that slows you down without providing real benefit. Most digital tools have good search, which is often faster than folder navigation anyway.

3. You run the "memory test" on a note you took 4 weeks ago and discover it says something you had completely forgotten. What does this tell you?

A Your memory is weak — consider taking fewer notes so each one gets more attention
B The topic wasn't important enough to remember — archive the note
C Your processing was insufficient — the note captured the idea but didn't consolidate it into memory, likely because you didn't process deeply enough or review it within the forgetting curve window
D The system is working — your notes are storing what your brain can't
Correct. A note that surprises you is a note that wasn't processed deeply enough — or that wasn't reviewed within the forgetting curve window. The fix is more deliberate Level 2/3 processing and a first spaced repetition review within a few days of capture.
If the note surprises you, your processing was insufficient. The goal isn't to have notes store what your memory doesn't — it's to have notes reinforce memory. A good system means the notes remind you of something you almost remember. Complete amnesia means the processing chain broke somewhere.
